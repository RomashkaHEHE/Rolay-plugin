# Context Index

This folder exists for design intent, priorities, and rationale.

Canonical docs elsewhere in the repo explain facts:

- what the protocol is
- what endpoints exist
- what modules do

The files here explain:

- why the project is shaped this way
- what goals a future change must preserve
- which tradeoffs were deliberate rather than accidental

Use these files when you need to change the system without blindly preserving every current implementation detail.
